  <p>code के एक ब्लॉक को दोहराने का एक तरीका।</p>

<p><code>जबकि</code> लूप कथनों को दोहराने में सहायक होते हैं जबकि कोई शर्त <code>सत्य</code> होती है। वे <code>if</code> कथनों की तरह हैं जो दोहराए जाते हैं। उदाहरण के लिए, एक <code>while</code> लूप इस प्रकार "कई रेखाएँ खींचें" विचार को व्यक्त करना आसान बनाता है:</p>
   <code>

   // पुनरावृत्ति पर नज़र रखने के लिए एक चर घोषित करें।
   मान लीजिए x = 10;

   <p>// x < 100 तक दोहराएँ
   जबकि (x < 100) {
     लाइन(x, 25, x, 75);</p>
   <p> // 20 तक वृद्धि।
     एक्स +=20;
   }
   </p></code>

   <p>लूप का हेडर कीवर्ड <code>जबकि</code> से शुरू होता है। लूप आम तौर पर दोहराए जाने या दोहराए जाने पर गिनते हैं या गिनते हैं। कोष्ठक में दिया गया कथन <code>x < 100</code> एक ऐसी स्थिति है जिसे लूप हर बार पुनरावृत्त होने पर जाँचता है। यदि स्थिति <code>true</code> है, तो लूप कर्ली के बीच code चलाता है
   ब्रेसिज़ <code>{}</code>, घुंघराले ब्रेसिज़ के बीच के code को लूप की बॉडी कहा जाता है। यदि स्थिति <code>गलत</code> है, तो बॉडी को छोड़ दिया जाता है और लूप बंद कर दिया जाता है।</p>

   <p>गलती से अनंत लूप बनाना आम बात है। उदाहरण के लिए,
  निम्नलिखित लूप कभी भी पुनरावृत्त होना बंद नहीं करता क्योंकि इसकी गिनती नहीं होती:</p>
  <code>

  // पुनरावृत्ति पर नज़र रखने के लिए एक चर घोषित करें।
मान लीजिए x = 10;
<p>// x < 100 तक दोहराएँ

   जबकि (x < 100) {
     पंक्ति(x, 25, x, 75);
   }</p>

   <p>// यह लूप की बॉडी में होना चाहिए!
   एक्स +=20;
   </p></code>

   <p>कथन <code>x += 20</code> लूप की बॉडी के बाद दिखाई देता है। इसका मतलब है कि वेरिएबल <code>x</code> <code>10</code> पर अटका हुआ है, जो हमेशा <code>100</code> से कम होता है।</p>
  <p><code>जबकि</code> लूप तब उपयोगी होते हैं जब पुनरावृत्तियों की संख्या पहले से ज्ञात नहीं होती है। उदाहरण के लिए, संकेंद्रित वृत्तों को यादृच्छिक वृद्धि पर खींचा जा सकता है:</p> <code>

   मान लीजिए d = 100;
   मान लीजिए न्यूनतम आकार = 5;

   <p>जबकि (डी > छोटा आकार) {
     वृत्त(50, 50, डी);
     डी -= यादृच्छिक(10);
   }

   </p></code>
