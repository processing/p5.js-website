description: >
  <p>पुनरावृत्तियों की संख्या ज्ञात होने पर code के एक ब्लॉक को दोहराने का एक तरीका।</p>

  <p><code>for</code> लूप कथनों को एक निश्चित संख्या में दोहराने में सहायक होते हैं। उदाहरण के लिए, एक <code>for</code> लूप इस प्रकार "पांच रेखाएं बनाएं" विचार को व्यक्त करना आसान बनाता है:</p>

     <code>
    for (let x = 10; x < 100; x += 20) {
      line(x, 25, x, 75);
    }
  </code>

  <p>लूप का हेडर कीवर्ड <code>for</code> से शुरू होता है। लूप आम तौर पर दोहराए जाने या दोहराए जाने पर गिनते हैं या गिनते हैं। कोष्ठक में दिए गए कथन <code>चलो x = 10; x < 100; x += 20</code> लूप को बताएं कि इसे कैसे दोहराया जाना चाहिए:</p>

   <ul>

   <li><code>let x = 10</code> लूप को गिनती शुरू करने के लिए कहता है
   <code>10</code> और वेरिएबल का उपयोग करके पुनरावृत्तियों पर नज़र रखें
   <code>x</code>.</li>

   <li><code>x < 100</code> लूप को <code>100</code> तक गिनने के लिए कहता है, लेकिन इसमें शामिल नहीं है।</li>

  <li><code>x += 20</code> प्रत्येक पुनरावृत्ति के अंत में लूप को <code>20</code> तक गिनने के लिए कहता है।</li></ul>

  <p>घुंघराले ब्रेसिज़ के बीच का code <code>{}</code> लूप का शरीर है। लूप बॉडी में कथन लूप के प्रत्येक पुनरावृत्ति के दौरान दोहराए जाते हैं।</p>

   <p>गलती से अनंत लूप बनाना आम बात है। जब ऐसा होता है, तो स्केच अनुत्तरदायी हो सकते हैं और वेब ब्राउज़र एक चेतावनी प्रदर्शित कर सकता है। उदाहरण के लिए, निम्न लूप कभी भी पुनरावृत्त होना बंद नहीं करता क्योंकि इसकी गिनती नहीं होती:</p>

     <code>
  for (let x = 10; x < 100; x = 20) {
    line(x, 25, x, 75);
  }
  </code>


  <p>कथन <code>x = 20</code> वेरिएबल <code>x</code> को <code>20</code> पर अटकाए रखता है, जो हमेशा <code>100</code> से कम होता है .</p> <p><code>for</code> लूप्स की उल्टी गिनती भी की जा सकती है:</p> <code>

    for (let d = 100; d > 0; d -= 10) {
    circle(50, 50, d);
  }

   </code>


   <p><code>for</code> लूप में अन्य लूप भी हो सकते हैं। निम्नलिखित नेस्टेड लूप बिंदुओं का एक ग्रिड बनाता है:</p> <code>

   // बाएं से दाएं लूप करें।
  for (let x = 10; x < 100; x += 10) {
  <p>  // Loop from top to bottom.
    for (let y = 10; y < 100; y += 10) {
      point(x, y);
    }</p>
  <p>}


   </p></code>

   <p><code>for</code> लूप किसी सरणी के तत्वों के माध्यम से पुनरावृत्ति करने में भी सहायक होते हैं। उदाहरण के लिए, एक सरणी के माध्यम से पुनरावृत्त करना आम बात है जिसमें कहां या क्या आकर्षित करना है इसके बारे में जानकारी होती है:</p>
   <code>

   // एक्स-निर्देशांक की एक सरणी बनाएं।

   let xCoordinates = [20, 40, 60];

  <p>for (let i = 0; i < xCoordinates.length; i += 1) {
    // Update the element.
    xCoordinates[i] += random(-1, 1);</p>
  <p>  // Draw a circle.
    circle(xCoordinates[i], 50, 20);
  }
  </p></code>

   <p>यदि सरणी के मान संशोधित नहीं किए गए हैं, तो <code>for...of</code> कथन code को सरल बना सकता है। वे पायथन में <code>for</code> लूप और C++ और Java में <code>for-each</code> लूप के समान हैं। निम्नलिखित लूप का प्रभाव समान है:</p>

   <code>

   // लूप के साथ वृत्त बनाएं।
    let xCoordinates = [20, 40, 60];

  <p>for (let i = 0; i < xCoordinates.length; i += 1) {
    circle(xCoordinates[i], 50, 20);
  }
  </p></code>

   <code>
   // कथन के लिए...के साथ वृत्त बनाएं।
     let xCoordinates = [20, 40, 60];

   <p>for (let x of xCoordinates) {
    circle(x, 50, 20);
   }

   </p></code>

   <p>उपरोक्त code स्निपेट में, वेरिएबल <code>i</code> और <code>x</code>
   अलग-अलग भूमिकाएँ हैं।</p>

   <p>पहले स्निपेट में, <code>i</code> की गिनती <code>0</code> से <code>2</code> तक होती है, जो कि इससे एक कम है

   <code>xनिर्देशांक.लंबाई</code>। <code>i</code> का उपयोग इंडेक्स <code>i</code> पर <code>xCodependents</code> में तत्व तक पहुंचने के लिए किया जाता है।</p>

   <p>दूसरे code स्निपेट में, <code>x</code> लूप की प्रगति या इंडेक्स का ट्रैक नहीं रख रहा है। प्रत्येक पुनरावृत्ति के दौरान, <code>x</code> में <code>xCodependents</code> का अगला तत्व शामिल होता है। <code>x</code>, <code>xनिर्देशांक</code> (<code>20</code>) की शुरुआत से शुरू होता है और इसके मान को <code>40</code> और फिर <code>60 पर अपडेट करता है। </code> अगले पुनरावृत्तियों के दौरान।</p>
