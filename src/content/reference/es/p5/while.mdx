  <p>Una forma de repetir un bloque de código.</p>
  <p>Los bucles <code>while</code> son útiles para repetir declaraciones mientras una condición sea <code>true</code>. Son como declaraciones <code>if</code> que se repiten. Por ejemplo, un bucle <code>while</code> facilita expresar la idea "dibujar varias líneas" de la siguiente manera:</p>
  <code>
    // Declare a variable to keep track of iteration.
    let x = 10;
    <p>// Repeat as long as x < 100
    while (x < 100) {
        line(x, 25, x, 75);
        </p>
        <p>// Increment by 20.
        x += 20;
    } </p>
  </code>

  <p>El encabezado del bucle comienza con la palabra clave <code>while</code>. Los bucles generalmente cuentan hacia arriba o hacia abajo mientras se repiten, o iteran. La declaración entre paréntesis <code>x < 100</code> es una condición que el bucle verifica cada vez que itera. Si la condición es <code>true</code>, el bucle ejecuta el código entre las llaves <code>{}</code>. El código entre las llaves corresponde al cuerpo del bucle. Si la condición es <code>false</code>, se salta el cuerpo y se detiene el bucle.</p>

  <p>Es común crear bucles infinitos accidentalmente. Por ejemplo, el siguiente bucle nunca deja de iterar porque no cuenta hacia arriba:</p>
  <code>
    // Declare a variable to keep track of iteration.
    let x = 10;
    <p>// Repeat as long as x < 100
    while (x < 100) {
        line(x, 25, x, 75);
    } </p>
    <p>// This should be in the loop's body!
    x += 20;
    </p>
  </code>

  <p>La declaración <code>x += 20</code> aparece después del cuerpo del bucle. Eso significa que la variable <code>x</code> está atascada en <code>10</code>, que siempre es menor que <code>100</code>.</p>

  <p>Los bucles <code>while</code> son útiles cuando el número de iteraciones no se conoce de antemano. Por ejemplo, se podrían dibujar círculos concéntricos en incrementos aleatorios:</p>
  <code>
    let d = 100;
    let minSize = 5;
    <p>while (d > minSize) {
        circle(50, 50, d);
        d -= random(10);
    } </p>
  </code>
