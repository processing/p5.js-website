description: |-
  <p>Captura una secuencia de fotogramas del lienzo que pueden ser guardados como imágenes.</p>

  <p><code>saveFrames()</code> crea un array de objetos de fotograma. Cada fotograma se almacena como un objeto con su tipo de archivo, nombre de archivo y datos de imagen como una cadena. Por ejemplo, el primer fotograma guardado podría tener las siguientes propiedades:</p>

  <p><code>{ ext: "'png', filenmame: 'frame0', imageData: 'data:image/octet-stream;base64, abc123' }</code>.</p>"

  "<p>El primer parámetro, <code>filename</code>, establece el prefijo para los nombres de archivo. Por ejemplo, configurar el prefijo como <code>'frame'</code> generaría los archivos de imagen <code>frame0.png</code>, <code>frame1.png</code>, y así sucesivamente.</p>"

  "<p>El segundo parámetro, <code>extension</code>, establece el tipo de archivo ya sea <code>'png'</code> o <code>'jpg'</code>.</p>"

  <p>El tercer parámetro, <code>duration</code>, establece la duración a grabar en segundos. La duración máxima es de 15 segundos.</p>

  <p>El cuarto parámetro, <code>framerate</code>, establece el número de fotogramas a grabar por segundo. El valor máximo de velocidad de fotogramas es 22. Se aplican límites a <code>duration</code> y <code>framerate</code> para evitar el uso excesivo de memoria. La grabación de lienzos grandes puede hacer que los bocetos se bloqueen fácilmente o incluso los navegadores web.</p>

  <p>El quinto parámetro, <code>callback</code>, es opcional. Si se pasa una función, los archivos de imagen no se guardarán por defecto. La función <em>callback</em> puede ser utilizada para procesar un array que contiene los datos de cada fotograma capturado. El array de datos de imagen contiene una secuencia de objetos con tres propiedades para cada fotograma: <code>imageData</code>, <code>filename</code>, y <code>extension</code>.</p>

  <p>Nota: Los fotogramas se descargan como archivos de imagen individuales por defecto.</p>
