description: >
  <p>Rota el sistema de coordenadas.</p>

  <p>De forma predeterminada, el eje x positivo apunta hacia la derecha y el eje y positivo

  apunta hacia abajo. La función <code>rotate()</code> cambia esta 
  orientación

  rotando el sistema de coordenadas alrededor del origen. Todo lo dibujado después

  de llamar a <code>rotate()</code> aparecerá rotado.</p>

  <p>El primer parámetro, <code>angle</code>, es la cantidad a rotar. Por
  ejemplo, llamar a

  <code>rotate(1)</code> rota el sistema de coordenadas en sentido horario 1 radián que 
  es

  casi 57˚. <code>rotate()</code> interpreta los valores de ángulo usando el

  <a href="/reference/p5/angleMode">angleMode()</a> actual.</p>

  <p>El segundo parámetro, <code>axis</code>, es opcional. Se utiliza para orientar
  rotaciones en 3D

  en modo WebGL. Si se pasa un <a href="/reference/p5/p5.Vector">p5.Vector</a>, 
  como en

  <code>rotate(QUARTER_PI, miVector)</code>, entonces el sistema de coordenadas se
  rotará

  <code>QUARTER_PI</code> radianes alrededor de <code>myVector</code>. Si se pasa un arreglo 
  de componentes 
  
  de vector, como en <code>rotate(QUARTER_PI, [1, 0, 0])</code>, entonces el sistema 
  de coordenadas
  
  se rotará <code>QUARTER_PI</code> radianes alrededor de un vector con los componentes

  <code>[1, 0, 0]</code>.</p>

  <p>De forma predeterminada, las transformaciones se acumulan. Por ejemplo, llamar
  a <code>rotate(1)</code>

  dos veces tiene el mismo efecto que llamar a <code>rotate(2)</code> una vez. Las

  funciones <a href="/reference/p5/push">push()</a> y <a
  href="/reference/p5/pop">pop()</a>

  se pueden usar para aislar transformaciones dentro de grupos de dibujo distintos.</p>

  <p>Nota: Las transformaciones se restablecen al principio del bucle de dibujo. Llamar

  a <code>rotate(1)</code> dentro de la función <a href="/reference/p5/draw">draw()</a>
  no hará que

  las formas giren.</p>
