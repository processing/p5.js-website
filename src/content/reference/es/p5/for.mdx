description: >
  <p>Una forma de repetir un bloque de código cuando se conoce el número de iteraciones.</p>
  <p>Los bucles <code>for</code> son útiles para repetir declaraciones un cierto número de veces. Por ejemplo, un bucle <code>for</code> facilita expresar la idea "dibujar cinco líneas" de la siguiente manera:</p>
  <code>
  for (let x = 10; x < 100; x += 20) {
  line(x, 25, x, 75);
  }
  </code>
  <p>La cabecera del bucle comienza con la palabra clave <code>for</code>. Los bucles generalmente incrementan o disminuyen a medida que se repiten, o iteran. Las declaraciones entre paréntesis <code>let x = 10; x < 100; x += 20</code> indican al bucle cómo debe repetirse:</p>
  <ul>
    <li><code>let x = 10</code> indica al bucle que comience a contar desde <code>10</code> y llevando un seguimiento de las iteraciones usando la variable <code>x</code>.</li>
    <li><code>x < 100</code> le indica al bucle que cuente hasta, pero sin incluir, <code>100</code>.</li>
    <li><code>x += 20</code> indica al bucle que incremente en <code>20</code> al final de cada iteración.</li>
  </ul>
  <p>El código entre llaves <code>{}</code> es el cuerpo del bucle. Las declaraciones dentro del cuerpo del bucle se repiten durante cada iteración del bucle.</p>
  <p>Es común crear bucles infinitos accidentalmente. Cuando esto sucede, los bocetos pueden volverse no responsivos y el navegador web puede mostrar una advertencia. Por ejemplo, el siguiente bucle nunca deja de iterar porque no se le ha indicado que incremente:</p>
  <code>
    for (let x = 10; x < 100; x = 20) {
    line(x, 25, x, 75);
  }
  </code>
  <p>La declaración <code>x = 20</code> mantiene la variable <code>x</code> estancada en <code>20</code>. El bucle continua infinitamente precisamente porque <code>20</code> siempre va a ser menor que <code>100</code>.</p>
  <p>Los bucles <code>for</code> también pueden contar hacia abajo (disminuir):</p>
  <code>
    for (let d = 100; d > 0; d -= 10) {
    circle(50, 50, d);
  }
  </code>
  <p>Los bucles <code>for</code> también pueden contener otros bucles. El siguiente bucle anidado dibuja una cuadrícula de puntos:</p>
  <code>
  // Loop from left to right
  for (let x = 10; x < 100; x += 10) {
  // Loop from top to bottom
  for (let y = 10; y < 100; y += 10) {
    point(x, y);
  }
  }
  </code>
  <p>Los bucles <code>for</code> también son útiles para iterar a través de los elementos de un arreglo. Por ejemplo, es común iterar a través de un arreglo que contiene información sobre dónde o qué dibujar:</p>

  <code>
  // Create an array of x-coordinates.Let xCoordinates = [20, 40, 60];

  for (let i = 0; i < xCoordinates.length; i += 1) {
  // Update the element.
  xCoordinates[i] += random(-1, 1);
  // Draw a circle.
  circle(xCoordinates[i], 50, 20);
  }
  </code>
  <p>Si los valores del array no se modifican, la declaración <code>for...of</code> puede simplificar el código. Son similares a los bucles <code>for</code> en Python y los bucles <code>for-each</code> en C++ y Java. Los siguientes bucles tienen el mismo efecto:</p>
  <code>
  //  Draw circles with a for loop
  let xCoordinates = [20, 40, 60];
  for (let i = 0; i < xCoordinates.length; i += 1) {
    circle(xCoordinates [i], 50, 20);
  }
  </code>
  <code>
  // Draw circles with a for...of statement.
  let xCoordinates = [20, 40, 60];
    for (let x of xCoordinates) {
    circle(x, 50, 20);
  }
  </code>
  <p>En estos fragmentos de código, las variables <code>i</code> y <code>x</code> tienen roles diferentes.</p>
  <p>En el primer fragmento, <code>i</code> cuenta desde <code>0</code> hasta <code>2</code>, que es uno menos que <code> xCoordinates.length</code>. <code>i</code> se utiliza para acceder al elemento en <code> xCoordinates </code> en el índice <code>i</code>.</p>
  <p>En el segundo fragmento de código, <code>x</code> no realiza un seguimiento del progreso del bucle o de un índice. Durante cada iteración, <code>x</code> contiene el siguiente elemento de <code> xCoordinates </code>. <code>x</code> comienza desde el principio de <code> xCoordinates </code> (<code>20</code>) y actualiza su valor a <code>40</code> y luego a <code>60</code> durante las siguientes iteraciones.</p>
