description: >
  <p>이미지에 이미지 필터를 적용합니다.</p>
      
      <p>미리 정의된 옵션은 다음과 같습니다:</p>
      
      <p><code>INVERT</code>
      
      이미지의 색상을 반전시킵니다. 매개변수를 사용하지 않습니다.</p>
      
      <p><code>GRAY</code>
      
      이미지를 그레이스케일로 변환합니다. 매개변수를 사용하지 않습니다.</p>
      
      <p><code>THRESHOLD</code>
      
      이미지를 흑백으로 변환합니다.
      
      임계값을 초과하는 그레이스케일 값의 픽셀은 흰색이 됩니다.
      
      나머지는 검정색으로 변환됩니다.
      
      임계값은 0.0(검은색)과 1.0(흰색) 사이여야 합니다.
      
      값이 지정되지 않은 경우, 0.5가 사용됩니다.</p>
      
      <p><code>OPAQUE</code>
      
      알파 채널을 완전히 불투명하게 설정합니다. 매개변수를 사용하지 않습니다.</p>
      
      <p><code>POSTERIZE</code>
      
      이미지의 색상 수를 제한합니다.
      
      각 색상 채널은 지정된 색상 수로 제한됩니다.
      
      2에서 255 사이의 값이 가능하지만, 결과는 낮은 값에서 가장 뚜렷하게 나타납니다.
      
      기본값은 4입니다.</p>
      
      <p><code>BLUR</code>
      
      이미지를 흐리게 만듭니다. 흐림 반경으로 흐림의 정도를 지정합니다.
      
      값이 클수록 흐림이 증가합니다. 기본값은 4입니다.
      
      <code>P2D</code> 모드에서는 가우시안 블러를 사용합니다.
      
      <code>WEBGL</code> 모드에서는 박스 블러를 사용합니다.</p>
      
      <p><code>ERODE</code>
      
      밝은 영역을 줄입니다. 매개변수를 사용하지 않습니다.</p>
      
      <p><code>DILATE</code>
      
      밝은 영역을 늘립니다. 매개변수를 사용하지 않습니다.</p>
      
      <p><code>filter()</code>는 백그라운드에서 기본적으로 빠른 WebGL을 사용합니다.
      
      <code>filter(BLUR, false)</code>와 같이 <code>P2D</code> 모드에서

      <code>false</code> 인수를 추가하여 비활성화할 수 있습니다.
      
      이것은 GPU 연산을 막거나 WebGL 미지원을 해결하기 위해 유용합니다.</p>
      
      <p>WebGL 모드에서 <code>filter()</code>는 사용자 정의 쉐이더도 사용할 수 있습니다.
      
      자세한 정보는 <a href="/reference/p5/createFilterShader">createFilterShader()</a>를 참조하십시오.</p>
