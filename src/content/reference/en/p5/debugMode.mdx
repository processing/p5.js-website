description: >
  <p>Adds a grid and an axes icon to clarify orientation in 3D sketches.</p>

  <p><code>debugMode()</code> adds a grid that shows where the “ground” is in a
  sketch. By

  default, the grid will run through the origin <code>(0, 0, 0)</code> of the
  sketch

  along the XZ plane. <code>debugMode()</code> also adds an axes icon that
  points along

  the positive x-, y-, and z-axes. Calling <code>debugMode()</code> displays the
  grid

  and axes icon with their default size and position.</p>

  <p>There are four ways to call <code>debugMode()</code> with optional
  parameters to

  customize the debugging environment.</p>

  <p>The first way to call <code>debugMode()</code> has one parameter,
  <code>mode</code>. If the

  system constant <code>GRID</code> is passed, as in
  <code>debugMode(GRID)</code>, then the grid

  will be displayed and the axes icon will be hidden. If the constant
  <code>AXES</code>

  is passed, as in <code>debugMode(AXES)</code>, then the axes icon will be
  displayed

  and the grid will be hidden.</p>

  <p>The second way to call <code>debugMode()</code> has six parameters. The
  first

  parameter, <code>mode</code>, selects either <code>GRID</code> or
  <code>AXES</code> to be displayed. The

  next five parameters, <code>gridSize</code>, <code>gridDivisions</code>,
  <code>xOff</code>, <code>yOff</code>, and

  <code>zOff</code> are optional. They’re numbers that set the appearance of the
  grid

  (<code>gridSize</code> and <code>gridDivisions</code>) and the placement of
  the axes icon

  (<code>xOff</code>, <code>yOff</code>, and <code>zOff</code>). For example,
  calling

  <code>debugMode(20, 5, 10, 10, 10)</code> sets the <code>gridSize</code> to 20
  pixels, the number

  of <code>gridDivisions</code> to 5, and offsets the axes icon by 10 pixels
  along the

  x-, y-, and z-axes.</p>

  <p>The third way to call <code>debugMode()</code> has five parameters. The
  first

  parameter, <code>mode</code>, selects either <code>GRID</code> or
  <code>AXES</code> to be displayed. The

  next four parameters, <code>axesSize</code>, <code>xOff</code>,
  <code>yOff</code>, and <code>zOff</code> are optional.

  They’re numbers that set the appearance of the size of the axes icon

  (<code>axesSize</code>) and its placement (<code>xOff</code>,
  <code>yOff</code>, and <code>zOff</code>).</p>

  <p>The fourth way to call <code>debugMode()</code> has nine optional
  parameters. The

  first five parameters, <code>gridSize</code>, <code>gridDivisions</code>,
  <code>gridXOff</code>, <code>gridYOff</code>,

  and <code>gridZOff</code> are numbers that set the appearance of the grid. For

  example, calling <code>debugMode(100, 5, 0, 0, 0)</code> sets the
  <code>gridSize</code> to 100,

  the number of <code>gridDivisions</code> to 5, and sets all the offsets to 0
  so that

  the grid is centered at the origin. The next four parameters,
  <code>axesSize</code>,

  <code>xOff</code>, <code>yOff</code>, and <code>zOff</code> are numbers that
  set the appearance of the size

  of the axes icon (<code>axesSize</code>) and its placement
  (<code>axesXOff</code>, <code>axesYOff</code>,

  and <code>axesZOff</code>). For example, calling

  <code>debugMode(100, 5, 0, 0, 0, 50, 10, 10, 10)</code> sets the
  <code>gridSize</code> to 100,

  the number of <code>gridDivisions</code> to 5, and sets all the offsets to 0
  so that

  the grid is centered at the origin. It then sets the <code>axesSize</code> to
  50 and

  offsets the icon 10 pixels along each axis.</p>
