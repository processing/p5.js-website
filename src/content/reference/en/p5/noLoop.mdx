description: >
  <p>Stops p5.js from continuously executing the code within <a
  href="/reference/p5/draw">draw()</a>.

  If <a href="/reference/p5/loop">loop()</a> is called, the code in <a
  href="/reference/p5/draw">draw()</a>

  begins to run continuously again. If using <a
  href="/reference/p5/noLoop">noLoop()</a>

  in <a href="/reference/p5/setup">setup()</a>, it should be the last line
  inside the block.</p>

  <p>When <a href="/reference/p5/noLoop">noLoop()</a> is used, it's not possible
  to manipulate

  or access the screen inside event handling functions such as

  <a href="/reference/p5/mousePressed">mousePressed()</a> or

  <a href="/reference/p5/keyPressed">keyPressed()</a>. Instead, use those
  functions to

  call <a href="/reference/p5/redraw">redraw()</a> or <a
  href="/reference/p5/loop">loop()</a>,

  which will run <a href="/reference/p5/draw">draw()</a>, which can update the
  screen

  properly. This means that when <a href="/reference/p5/noLoop">noLoop()</a> has
  been

  called, no drawing can happen, and functions like <a
  href="/reference/p5/saveFrames">saveFrames()</a>

  or <a href="/reference/p5/loadPixels">loadPixels()</a> may not be used.</p>

  <p>Note that if the sketch is resized, <a
  href="/reference/p5/redraw">redraw()</a> will

  be called to update the sketch, even after <a
  href="/reference/p5/noLoop">noLoop()</a>

  has been specified. Otherwise, the sketch would enter an odd state until

  <a href="/reference/p5/loop">loop()</a> was called.</p>

  <p>Use <a href="/reference/p5/isLooping">isLooping()</a> to check the current
  state of <a href="/reference/p5/loop">loop()</a>.</p>

    function setup() {

    }
    </code>

    let x = 0;
    function setup() {
      createCanvas(100, 100);
    }

    function draw() {
      background(204);
      x = x + 0.1;
      if (x > width) {
        x = 0;
      }
      line(x, 0, x, height);
    }

    function mousePressed() {
      noLoop();
    }

    function mouseReleased() {
      loop();
    }
    </code>
