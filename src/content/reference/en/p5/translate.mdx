description: >
  <p>Translates the coordinate system.</p>

  <p>By default, the origin <code>(0, 0)</code> is at the sketch's top-left
  corner in 2D

  mode and center in WebGL mode. The <code>translate()</code> function shifts
  the origin

  to a different position. Everything drawn after <code>translate()</code> is
  called

  will appear to be shifted. There are two ways to call <code>translate()</code>
  with

  parameters that set the origin's position.</p>

  <p>The first way to call <code>translate()</code> uses numbers to set the
  amount of

  translation. The first two parameters, <code>x</code> and <code>y</code>, set
  the amount to

  translate along the positive x- and y-axes. For example, calling

  <code>translate(20, 30)</code> translates the origin 20 pixels along the
  x-axis and 30

  pixels along the y-axis. The third parameter, <code>z</code>, is optional. It
  sets the

  amount to translate along the positive z-axis. For example, calling

  <code>translate(20, 30, 40)</code> translates the origin 20 pixels along the
  x-axis,

  30 pixels along the y-axis, and 40 pixels along the z-axis.</p>

  <p>The second way to call <code>translate()</code> uses a

  <a href="/reference/p5/p5.Vector">p5.Vector</a> object to set the amount of

  translation. For example, calling <code>translate(myVector)</code> uses the
  x-, y-,

  and z-components of <code>myVector</code> to set the amount to translate along
  the x-,

  y-, and z-axes. Doing so is the same as calling

  <code>translate(myVector.x, myVector.y, myVector.z)</code>.</p>

  <p>By default, transformations accumulate. For example, calling

  <code>translate(10, 0)</code> twice has the same effect as calling

  <code>translate(20, 0)</code> once. The <a
  href="/reference/p5/push/">push()</a> and

  <a href="/reference/p5/pop/">pop()</a> functions can be used to isolate

  transformations within distinct drawing groups.</p>

  <p>Note: Transformations are reset at the beginning of the draw loop. Calling

  <code>translate(10, 0)</code> inside the <a
  href="/reference/p5/draw/">draw()</a> function won't

  cause shapes to move continuously.</p>
