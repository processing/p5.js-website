description: >
  <p>Changes the coordinate system used for textures when they’re applied to

  custom shapes.</p>

  <p>In order for <a href="/reference/p5/texture/">texture()</a> to work, a
  shape needs a

  way to map the points on its surface to the pixels in an image. Built-in

  shapes such as <a href="/reference/p5/rect/">rect()</a> and

  <a href="/reference/p5/box/">box()</a> already have these texture mappings
  based on

  their vertices. Custom shapes created with

  <a href="/reference/p5/vertex/">vertex()</a> require texture mappings to be
  passed as

  uv coordinates.</p>

  <p>Each call to <a href="/reference/p5/vertex/">vertex()</a> must include 5
  arguments,

  as in <code>vertex(x, y, z, u, v)</code>, to map the vertex at coordinates
  <code>(x, y, z)</code>

  to the pixel at coordinates <code>(u, v)</code> within an image. For example,
  the

  corners of a rectangular image are mapped to the corners of a rectangle by
  default:</p>

  <code>

  // Apply the image as a texture.

  texture(img);


  <p>// Draw the rectangle.

  rect(0, 0, 30, 50);

  </p></code>

  <p>If the image in the code snippet above has dimensions of 300 x 500 pixels,

  the same result could be achieved as follows:</p>

  <code>

  // Apply the image as a texture.

  texture(img);


  <p>// Draw the rectangle.

  beginShape();</p>

  <p>// Top-left.

  // u: 0, v: 0

  vertex(0, 0, 0, 0, 0);</p>

  <p>// Top-right.

  // u: 300, v: 0

  vertex(30, 0, 0, 300, 0);</p>

  <p>// Bottom-right.

  // u: 300, v: 500

  vertex(30, 50, 0, 300, 500);</p>

  <p>// Bottom-left.

  // u: 0, v: 500

  vertex(0, 50, 0, 0, 500);</p>

  <p>endShape();

  </p></code>

  <p><code>textureMode()</code> changes the coordinate system for uv
  coordinates.</p>

  <p>The parameter, <code>mode</code>, accepts two possible constants. If
  <code>NORMAL</code> is

  passed, as in <code>textureMode(NORMAL)</code>, then the texture’s uv
  coordinates can

  be provided in the range 0 to 1 instead of the image’s dimensions. This can

  be helpful for using the same code for multiple images of different sizes.

  For example, the code snippet above could be rewritten as follows:</p>

  <code>

  // Set the texture mode to use normalized coordinates.

  textureMode(NORMAL);


  <p>// Apply the image as a texture.

  texture(img);</p>

  <p>// Draw the rectangle.

  beginShape();</p>

  <p>// Top-left.

  // u: 0, v: 0

  vertex(0, 0, 0, 0, 0);</p>

  <p>// Top-right.

  // u: 1, v: 0

  vertex(30, 0, 0, 1, 0);</p>

  <p>// Bottom-right.

  // u: 1, v: 1

  vertex(30, 50, 0, 1, 1);</p>

  <p>// Bottom-left.

  // u: 0, v: 1

  vertex(0, 50, 0, 0, 1);</p>

  <p>endShape();

  </p></code>

  <p>By default, <code>mode</code> is <code>IMAGE</code>, which scales uv
  coordinates to the

  dimensions of the image. Calling <code>textureMode(IMAGE)</code> applies the
  default.</p>

  <p>Note: <code>textureMode()</code> can only be used in WebGL mode.</p>

    let img;

    // Load an image and create a p5.Image object.
    function preload() {
      img = loadImage('/assets/laDefense.jpg');
    }

    function setup() {
      createCanvas(100, 100, WEBGL);

      describe('An image of a ceiling against a black background.');
    }

    function draw() {
      background(0);

      // Apply the image as a texture.
      texture(img);

      // Draw the custom shape.
      // Use the image's width and height as uv coordinates.
      beginShape();
      vertex(-30, -30, 0, 0);
      vertex(30, -30, img.width, 0);
      vertex(30, 30, img.width, img.height);
      vertex(-30, 30, 0, img.height);
      endShape();
    }
    </code>
