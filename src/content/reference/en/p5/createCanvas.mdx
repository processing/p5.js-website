description: >
  <p>Creates a canvas element in the document and sets its dimensions

  in pixels. This method should be called only once at the start of <a
  href="/reference/p5/setup">setup()</a>.

  Calling <a href="/reference/p5/createCanvas">createCanvas</a> more than once
  in a

  sketch will result in very unpredictable behavior. If you want more than

  one drawing canvas you could use <a
  href="/reference/p5/createGraphics">createGraphics()</a>

  (hidden by default but it can be shown).</p>

  <p>Important note: in 2D mode (i.e. when <code>p5.Renderer</code> is not set)
  the origin (0,0)

  is positioned at the top left of the screen. In 3D mode (i.e. when
  <code>p5.Renderer</code>

  is set to <code>WEBGL</code>), the origin is positioned at the center of the
  canvas.

  See <a href="https://github.com/processing/p5.js/issues/1545">this issue</a>
  for more information.</p>

  <p>A WebGL canvas will use a WebGL2 context if it is supported by the browser.

  Check the <a href="/reference/p5/webglVersion">webglVersion</a> property to
  check what

  version is being used, or call <a
  href="/reference/p5/setAttributes">setAttributes({ version: 1 })</a>

  to create a WebGL1 context.</p>

  <p>The system variables width and height are set by the parameters passed to
  this

  function. If <a href="/reference/p5/createCanvas">createCanvas()</a> is not
  used, the

  window will be given a default size of 100Ã—100 pixels.</p>

  <p>Optionally, an existing canvas can be passed using a selector, ie.
  <code>document.getElementById('')</code>.

  If specified, avoid using <code>setAttributes()</code> afterwards, as this
  will remove and recreate the existing canvas.</p>

  <p>For more ways to position the canvas, see the

  <a href="https://github.com/processing/p5.js/wiki/Positioning-your-canvas">

  positioning the canvas</a> wiki page.</p>
