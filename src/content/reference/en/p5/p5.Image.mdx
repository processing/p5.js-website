description: >
  <p>A class to describe an image.</p>

  <p>Images are rectangular grids of pixels that can be displayed and
  modified.</p>

  <p>Existing images can be loaded by calling

  <a href="/reference/p5/loadImage/">loadImage()</a>. Blank images can be
  created by

  calling <a href="/reference/p5/createImage/">createImage()</a>.
  <code>p5.Image</code> objects

  have methods for common tasks such as applying filters and modifying

  pixel values.</p>

    <code>
    function setup() {
      createCanvas(100, 100);

      background(200);

      // Create a p5.Image object.
      let img = createImage(66, 66);

      // Load the image's pixels.
      img.loadPixels();

      // Set the pixels to black.
      for (let x = 0; x < img.width; x += 1) {
        for (let y = 0; y < img.height; y += 1) {
          img.set(x, y, 0);
        }
      }

      // Update the image.
      img.updatePixels();

      // Display the image.
      image(img, 17, 17);

      describe('A black square drawn in the middle of a gray square.');
    }
    </code>
    </div>

  updatePixels:
    description: >
      <p>Updates the canvas with the RGBA values in the

      <a href="/reference/p5.Image/pixels/">img.pixels</a> array.</p>

      <p><code>img.updatePixels()</code> only needs to be called after changing
      values in

      the <a href="/reference/p5.Image/pixels/">img.pixels</a> array. Such
      changes can be

      made directly after calling

      <a href="/reference/p5.Image/loadPixels/">img.loadPixels()</a> or by
      calling

      <a href="/reference/p5.Image/set/">img.set()</a>.</p>

      <p>The optional parameters <code>x</code>, <code>y</code>,
      <code>width</code>, and <code>height</code> define a

      subsection of the image to update. Doing so can improve performance in

      some cases.</p>

      <p>If the image was loaded from a GIF, then calling
      <code>img.updatePixels()</code>

      will update the pixels in current frame.</p>
    path: p5.Image/updatePixels
  get:
    description: >
      <p>Gets a pixel or a region of pixels from the image.</p>

      <p><code>img.get()</code> is easy to use but it's not as fast as

      <a href="/reference/p5.Image/pixels/">img.pixels</a>. Use

      <a href="/reference/p5.Image/pixels/">img.pixels</a> to read many pixel
      values.</p>

      <p>The version of <code>img.get()</code> with no parameters returns the
      entire image.</p>

      <p>The version of <code>img.get()</code> with two parameters, as in
      <code>img.get(10, 20)</code>,

      interprets them as coordinates. It returns an array with the

      <code>[R, G, B, A]</code> values of the pixel at the given point.</p>

      <p>The version of <code>img.get()</code> with four parameters, as in

      <code>img,get(10, 20, 50, 90)</code>, interprets them as

      coordinates and dimensions. The first two parameters are the coordinates

      of the upper-left corner of the subsection. The last two parameters are

      the width and height of the subsection. It returns a subsection of the

      canvas in a new <a href="/reference/p5/p5.Image">p5.Image</a> object.</p>

      <p>Use <code>img.get()</code> instead of <a
      href="/reference/p5/get/">get()</a> to work directly

      with images.</p>
    path: p5.Image/get
  set:
    description: >
      <p>Sets the color of one or more pixels within an image.</p>

      <p><code>img.set()</code> is easy to use but it's not as fast as

      <a href="/reference/p5.Image/pixels/">img.pixels</a>. Use

      <a href="/reference/p5.Image/pixels/">img.pixels</a> to set many pixel
      values.</p>

      <p><code>img.set()</code> interprets the first two parameters as x- and
      y-coordinates. It

      interprets the last parameter as a grayscale value, a <code>[R, G, B,
      A]</code> pixel

      array, a <a href="/reference/p5/p5.Color">p5.Color</a> object, or another

      <a href="/reference/p5/p5.Image">p5.Image</a> object.</p>

      <p><a href="/reference/p5.Image/updatePixels/">img.updatePixels()</a> must
      be called

      after using <code>img.set()</code> for changes to appear.</p>
    path: p5.Image/set

  copy:
    description: >
      <p>Copies pixels from a source image to this image.</p>

      <p>The first parameter, <code>srcImage</code>, is an optional

      <a href="/reference/p5/p5.Image">p5.Image</a> object to copy. If a source
      image isn't

      passed, then <code>img.copy()</code> can copy a region of this image to
      another

      region.</p>

      <p>The next four parameters, <code>sx</code>, <code>sy</code>,
      <code>sw</code>, and <code>sh</code> determine the region

      to copy from the source image. <code>(sx, sy)</code> is the top-left
      corner of the

      region. <code>sw</code> and <code>sh</code> are the region's width and
      height.</p>

      <p>The next four parameters, <code>dx</code>, <code>dy</code>,
      <code>dw</code>, and <code>dh</code> determine the region

      of this image to copy into. <code>(dx, dy)</code> is the top-left corner
      of the

      region. <code>dw</code> and <code>dh</code> are the region's width and
      height.</p>

      <p>Calling <code>img.copy()</code> will scale pixels from the source
      region if it isn't

      the same size as the destination region.</p>
    path: p5.Image/copy
  mask:
    description: >
      <p>Masks part of the image with another.</p>

      <p><code>img.mask()</code> uses another <a
      href="/reference/p5/p5.Image">p5.Image</a> object's

      alpha channel as the alpha channel for this image. Masks are cumulative

      and can't be removed once applied. If the mask has a different

  filter:
    description: >
      <p>Applies an image filter to the image.</p>

      <p>The preset options are:</p>

      <p><code>INVERT</code>

      Inverts the colors in the image. No parameter is used.</p>

      <p><code>GRAY</code>

      Converts the image to grayscale. No parameter is used.</p>

      <p><code>THRESHOLD</code>

      Converts the image to black and white. Pixels with a grayscale value

      above a given threshold are converted to white. The rest are converted to

      black. The threshold must be between 0.0 (black) and 1.0 (white). If no

      value is specified, 0.5 is used.</p>

      <p><code>OPAQUE</code>

      Sets the alpha channel to be entirely opaque. No parameter is used.</p>

      <p><code>POSTERIZE</code>

      Limits the number of colors in the image. Each color channel is limited to

      the number of colors specified. Values between 2 and 255 are valid, but

      results are most noticeable with lower values. The default value is 4.</p>

      <p><code>BLUR</code>

      Blurs the image. The level of blurring is specified by a blur radius.
      Larger

      values increase the blur. The default value is 4. A gaussian blur is used

      in <code>P2D</code> mode. A box blur is used in <code>WEBGL</code>
      mode.</p>

      <p><code>ERODE</code>

      Reduces the light areas. No parameter is used.</p>

      <p><code>DILATE</code>

      Increases the light areas. No parameter is used.</p>
    path: p5.Image/filter
  blend:

      <p>Copies a region of pixels from another image into this one.</p>

      <p>The first parameter, <code>srcImage</code>, is the

      <p>The next four parameters, <code>sx</code>, <code>sy</code>,
      <code>sw</code>, and <code>sh</code> determine the region

      to blend from the source image. <code>(sx, sy)</code> is the top-left
      corner of the

      region. <code>sw</code> and <code>sh</code> are the regions width and
      height.</p>

      <p>The next four parameters, <code>dx</code>, <code>dy</code>,
      <code>dw</code>, and <code>dh</code> determine the region

      of the canvas to blend into. <code>(dx, dy)</code> is the top-left corner
      of the

      region. <code>dw</code> and <code>dh</code> are the regions width and
      height.</p>

      <p>The tenth parameter, <code>blendMode</code>, sets the effect used to
      blend the images'

      colors. The options are <code>BLEND</code>, <code>DARKEST</code>,
      <code>LIGHTEST</code>, <code>DIFFERENCE</code>,

      <code>MULTIPLY</code>, <code>EXCLUSION</code>, <code>SCREEN</code>,
      <code>REPLACE</code>, <code>OVERLAY</code>, <code>HARD_LIGHT</code>,

      <code>SOFT_LIGHT</code>, <code>DODGE</code>, <code>BURN</code>,
      <code>ADD</code>, or <code>NORMAL</code>.</p>
    path: p5.Image/blend
  save:
    description: >
      <p>Saves the image to a file.</p>

      <p>By default, <code>img.save()</code> saves the image as a PNG image
      called

      <code>untitled.png</code>.</p>

      <p>The first parameter, <code>filename</code>, is optional. It's a string
      that sets the

      file's name. If a file extension is included, as in

      <code>img.save('drawing.png')</code>, then the image will be saved using
      that

      format.</p>

      <p>The second parameter, <code>extension</code>, is also optional. It sets
      the files format.

      Either <code>'png'</code> or <code>'jpg'</code> can be used. For example,
      <code>img.save('drawing', 'jpg')</code>

      saves the canvas to a file called <code>drawing.jpg</code>.</p>

      <p>Note: The browser will either save the file immediately or prompt the
      user

      with a dialogue window.</p>

      <p>The image will only be downloaded as an animated GIF if it was loaded

      from a GIF file. See <a href="/reference/p5/saveGif/">saveGif()</a> to
      create new

      GIFs.</p>

  pixels:
    description: >
      <p>An array containing the color of each pixel in the image.</p>

      <p>Colors are stored as numbers representing red, green, blue, and alpha

      (RGBA) values. <code>img.pixels</code> is a one-dimensional array for
      performance

      reasons.</p>

      <p>Each pixel occupies four elements in the pixels array, one for each

      RGBA value. For example, the pixel at coordinates (0, 0) stores its

      RGBA values at <code>img.pixels[0]</code>, <code>img.pixels[1]</code>,
      <code>img.pixels[2]</code>,

      and <code>img.pixels[3]</code>, respectively. The next pixel at
      coordinates (1, 0)

      stores its RGBA values at <code>img.pixels[4]</code>,
      <code>img.pixels[5]</code>,

      <code>img.pixels[6]</code>, and <code>img.pixels[7]</code>. And so on. The
      <code>img.pixels</code> array

      for a 100×100 <a href="/reference/p5/p5.Image">p5.Image</a> object has

      100 × 100 × 4 = 40,000 elements.</p>

      <p>Accessing the RGBA values for a pixel in the image requires a little

      math as shown in the examples below. The

      <a href="/reference/p5.Image/loadPixels/">img.loadPixels()</a>

      method must be called before accessing the <code>img.pixels</code> array.
      The

      <a href="/reference/p5.Image/updatePixels/">img.updatePixels()</a> method
      must be

      called after any changes are made.</p>
    path: p5.Image/pixels
