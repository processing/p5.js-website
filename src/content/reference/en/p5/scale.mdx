description: >
  <p>Scales the coordinate system.</p>

  <p>By default, shapes are drawn at their original scale. A rectangle that's 50

  pixels wide appears to take up half the width of a 100 pixel-wide canvas.

  The <code>scale()</code> function can shrink or stretch the coordinate system
  so that

  shapes appear at different sizes. There are two ways to call
  <code>scale()</code> with

  parameters that set the scale factor(s).</p>

  <p>The first way to call <code>scale()</code> uses numbers to set the amount
  of scaling.

  The first parameter, <code>s</code>, sets the amount to scale each axis. For
  example,

  calling <code>scale(2)</code> stretches the x-, y-, and z-axes by a factor of
  2. The

  next two parameters, <code>y</code> and <code>z</code>, are optional. They set
  the amount to

  scale the y- and z-axes. For example, calling <code>scale(2, 0.5, 1)</code>
  stretches

  the x-axis by a factor of 2, shrinks the y-axis by a factor of 0.5, and

  leaves the z-axis unchanged.</p>

  <p>The second way to call <code>scale()</code> uses a <a
  href="/reference/p5/p5.Vector">p5.Vector</a>

  object to set the scale factors. For example, calling
  <code>scale(myVector)</code>

  uses the x-, y-, and z-components of <code>myVector</code> to set the amount
  of

  scaling along the x-, y-, and z-axes. Doing so is the same as calling

  <code>scale(myVector.x, myVector.y, myVector.z)</code>.</p>

  <p>By default, transformations accumulate. For example, calling
  <code>scale(1)</code>

  twice has the same effect as calling <code>scale(2)</code> once. The

  <a href="/reference/p5/push/">push()</a> and <a
  href="/reference/p5/pop/">pop()</a> functions

  can be used to isolate transformations within distinct drawing groups.</p>

  <p>Note: Transformations are reset at the beginning of the draw loop. Calling

  <code>scale(2)</code> inside the <a href="/reference/p5/draw/">draw()</a>
  function won't cause

  shapes to grow continuously.</p>
